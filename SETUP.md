# Crystal Path — Project Setup

## Starting point

The repo already contained a bare Tauri + Vue 3 + TypeScript scaffold generated by `@tauri-apps/cli`. It had no router, no state management, and a single default `App.vue` with the Tauri "greet" demo.

---

## 1. Install dependencies

Added Vue Router 4 and Pinia to the existing project:

```bash
npm install vue-router@4 pinia
```

Also installed `playwright` locally for UI verification:

```bash
npm install playwright
```

---

## 2. Configure the `@` path alias

The Vite config had no alias for `@`, which broke all `@/...` imports. Added it to `vite.config.ts`:

```ts
import { fileURLToPath, URL } from "node:url";

resolve: {
  alias: {
    '@': fileURLToPath(new URL('./src', import.meta.url)),
  },
},
```

---

## 3. Global styles (`src/assets/main.css`)

Created from scratch with:

- Google Fonts import for **Press Start 2P** (the 8-bit pixel font)
- Universal box-sizing / margin / padding reset
- CSS custom properties for the full color palette:
  - Night/dark/panel/border/text/muted/gold shades for the UI chrome
  - Per-class accent colors: `--c-fighter`, `--c-thief`, `--c-wmage`, `--c-bmage`
- `body` set to the dark background and pixel font
- `#app` set to `min-height: 100vh` flex column

---

## 4. Data (`src/data/classes.js`)

Defined the `CLASSES` array — the single source of truth for all four RPG classes. Each entry contains:

| Field | Purpose |
|---|---|
| `id` | Route key / store identifier |
| `name` / `tag` | Display name and sub-title |
| `color` | Hex accent color used by components via CSS var |
| `description` | Flavour text |
| `ability` / `abilityDesc` | Class-specific navigation power |
| `stats` | `{ str, exp, agi }` values 1–5 for stat pip display |
| `sprite` | Array of strings encoding a pixel-art sprite |

Sprite encoding: each character in a row string maps to a color — `'1'` = class color, `'2'` = skin tone, `'w'` = white, `'g'` = gray, `'G'` = gold, `'d'` = dark, `'0'` = transparent.

---

## 5. Pinia store (`src/stores/player.js`)

Simple composition-style store with:

- `chosenClass` — the full class object selected by the user (null until chosen)
- `onboardingStep` — integer tracking progress through the onboarding flow
- `selectClass(cls)` — sets the chosen class
- `advanceOnboarding()` — increments the step counter
- `reset()` — clears both back to defaults

---

## 6. Router (`src/router/index.js`)

Two routes using `createWebHistory`:

| Path | Name | Component |
|---|---|---|
| `/` | `class-select` | `ClassSelectScreen.vue` |
| `/onboarding` | `onboarding` | `OnboardingScreen.vue` |

## 8. Components

### `PixelSprite.vue`

Canvas-based renderer for the sprite data in `classes.js`.

- Props: `rows` (Array), `color` (String), `pixelSize` (Number, default 4)
- On mount and on any prop change, redraws the canvas
- Maps each character to a fill color; skips `'0'` (transparent)
- Sets `image-rendering: pixelated` for crisp pixel scaling at any size

### `StarField.vue`

Animated twinkling star background.

- On mount, generates 80 stars with random position, size, animation delay, and duration
- Renders as absolutely-positioned `<div>` elements with a CSS `@keyframes twinkle` animation (opacity 0.15 → 1 → 0.15)
- `pointer-events: none` so it doesn't interfere with card clicks

### `PixelDivider.vue`

A row of small square dots used as a decorative section separator.

- Props: `count` (default 12), `color` (default `var(--ff-gold-dark)`)
- Each dot is a 4×4px inline-block `<span>`

### `ClassCard.vue`

The interactive class selection card.

- Props: `classData` (Object), `selected` (Boolean)
- Emits: `select`
- Sets `--cc` CSS variable on the root element to the class's accent color; all color-dependent rules (border, pips, ability bar) reference `var(--cc)` in scoped CSS
- Accessible: `role="radio"`, `aria-checked`, `tabindex="0"`, responds to Enter/Space keydown
- Stat pips: 3 rows × 5 pips; `.filled` pips get `background: var(--cc)` based on the class's stat value

---

## 9. Views

### `ClassSelectScreen.vue`

Main screen composing all components.

- Manages `selectedId` as local `ref`; commits to the Pinia store only on confirm
- Corner decoration divs (gold bracket corners, absolutely positioned)
- `StarField` behind all content via `position: relative / absolute`
- 4-column responsive grid (`auto-fit, minmax(200px, 1fr)`) of `ClassCard` components
- Confirm button: disabled until a class is selected; on click, looks up the full class object from `CLASSES`, calls `store.selectClass()`, then navigates to `/onboarding`
- "FIGHTER selected" confirmation text appears below the button once a class is chosen

### `OnboardingScreen.vue` (stub)

Placeholder that confirms the selection made it through the store:

```
Welcome, [ClassName].
Onboarding flow coming soon.
[← Back]
```

The Back button routes back to `/` via `router.push('/')`.
